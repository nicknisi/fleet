Phase 01 — Grouped dashboard & window-name labels

Context · 01

The problem

01The fleet dashboard labels each agent with a `session:window` string truncated at a fixed 15 characters, so with multiple agents per tmux session every row reads like `authkit:🤖 wor…` — the window name, the part that actually distinguishes agents, is the part that gets cut off (screenshot from 2026-06-10, 8 agents).

02The tmux second statusline renders the raw session name per chip (src/cli/status.ts:26), never adopting the window-aware labels shipped in v0.7.0 (PR #11), so two ready agents in the same session produce identical chips and the user can't tell which window needs attention.

Commit · 02

Goals

1 Dashboard rows are grouped under session header lines, with window names rendered in full (untruncated) at typical terminal widths.
2 Groups order by their most urgent member so attention-needing agents still bubble to the top; rows within a group order by urgency then window name.
3 Statusline chips show the window name (falling back to the session name when the window name is empty or equals the session name).
4 Preview header and kill confirmation use the same window-first labeling rule.
Signal · 03

Done when…

7 signals
  • 01 bun test passes, including new cases: a session with 2+ agents renders a header line plus indented agent rows; a single-agent session renders one inline `session · window` row with no header.
  • 02 Group ordering test: a session containing a PERMIT agent sorts above a session whose best status is DONE, regardless of alphabetical order. Within-group test: a PERMIT row renders above a DONE row in the same group; equal-status rows order by window name.
  • 03 Width test: at 120 and 80 columns, a 20-char window name renders untruncated; when narrowing, the detail column shrinks to its minimum and the branch column drops before the window name loses any character.
  • 04 Statusline test: formatStatusLine renders the window name per chip and falls back to the session name when the window is empty or equals the session name; click ranges (paneId) are unchanged.
  • 05 Preview header and kill confirmation tests assert the window-first label.
  • 06 bun run typecheck and bun run lint pass.
  • 07 Navigation (j/k) and selection skip header lines; the `/` filter hides groups with no matching rows.

MVP — must ship ×6
  • Grouped dashboard rendering: session header lines, 2-space-indented agent rows, inline singleton rows— The core fix for the unreadable name column
  • Urgency-bubbled group ordering; urgency-then-window ordering within groups— Preserves the dashboard's attention-first purpose under grouping
  • Content-sized window column with shrink order detail → branch → window last; all width math uses the existing visibleLength() helper so emoji-prefixed window names pad correctly— Implements 'never truncate the window' within finite terminal width without one-cell emoji misalignment
  • Statusline chips: window name with session fallback— Matches the user's mental model — chips answer 'which window do I jump to'
  • Preview header and kill confirmation adopt window-first labels— One consistent labeling rule app-wide
  • Navigation/selection skip headers; filter hides empty groups— Grouping must not break existing keyboard flow
Stretch — if time permits ×1
  • Header line shows per-session agent count and aggregate status icon
Out of scope — said no on purpose
  • Collapsible/expandable session groups — Interaction-model change beyond the readability fix; revisit if session counts grow
  • User-configurable sort modes — Urgency-bubbled grouping was chosen explicitly; no evidence a toggle is needed
  • Pane-level rows (multiple agents in one window) — State engine keys agents by pane already; the display unit chosen is the window
Future — someday, maybe
  • Collapsible session groups with persisted state
  • Session-scoped actions on header rows (kill all, ack all)
