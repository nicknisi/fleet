Implementation Notes — Phase 1

session-grouped-dashboard · 2026-06-10

Dashboard rendering — off-spec change to index.ts mouse-click mapping

Context
The spec's Modified Files list did not include index.ts, but the scout flagged that the click handler (index.ts:636-641) computed idx = mouse.y - headerHeight - 2 and indexed straight into visibleStates() — a 1:1 line-to-state assumption that grouped header lines break. It also ignored scroll offset (a pre-existing latent bug when the list scrolls).
Decision
Added an exported stateAtLine(app, lineIdx, maxRows) in dashboard.ts that maps a clicked line through dashboardRows() with the same calculateScroll the renderer uses, and rewired index.ts to use it. Header clicks select nothing; agent clicks resolve by paneId. This also fixes the pre-existing scroll-offset bug.
Alternative
Leaving index.ts untouched (strict spec adherence) — rejected because clicks would silently select the wrong agent, including acknowledging the wrong DONE agent.

Header row — singular/plural and alignment choices

Context
The spec defined the header as {icon} {session} · {count} agents but not its column alignment or pluralization.
Decision
Headers indent 2 spaces so the aggregate icon sits in the same column as agent-row status icons and the session name aligns with name cells. Count always reads "agents" — headers only exist for 2+ member groups, so the singular case is unreachable.
Alternative
Flush-left headers (as in the approved ASCII preview) — rejected in favor of icon-column alignment; the preview predates the stretch-scope aggregate icon.
